# nodeseek-sou

作者：[@mskatoni](https://github.com/mskatoni)  

一个低内存友好的 Tampermonkey / Violentmonkey 用户脚本，用于在 NodeSeek 列表页拉取当前页后续若干页帖子，并支持按浏览量、评论数排序，以及按发布时间、用户名、作者等级过滤。

许可证：AGPL-3.0-only

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 或 [Violentmonkey](https://violentmonkey.github.io/)。
2. 新建用户脚本。
3. 将 [`nodeseek-sou.user.js`](./nodeseek-sou.user.js) 的内容粘贴进去并保存。

如果你已经把项目推送到 GitHub，可以使用下面的 Raw 地址安装：

```text
https://raw.githubusercontent.com/mskatoni/nodeseek-sou/main/nodeseek-sou.user.js
```

## 功能

- 拉取当前 NodeSeek 列表页后续 `X` 页帖子。
- 每页请求前等待 5 秒，避免触发过快访问限制。
- 后续页记录会缓存到 IndexedDB，默认 30 分钟内重复查询可直接复用缓存。
- 支持 Web Worker 后台过滤和排序；Worker 不可用时自动回退主线程。
- 按浏览量从高到低排序。
- 按评论数从高到低排序。
- 只显示最近 `X` 天内发布的帖子。
- 只显示指定用户名发布的帖子，支持多个用户名，用空格、英文逗号、中文逗号分隔。
- Beta：隐藏作者等级小于等于指定阈值的帖子，例如填写 `2` 会隐藏识别为 Lv0、Lv1、Lv2 的作者帖子。
- 在当前筛选/排序结果中，按标题关键词统计作者命中帖数，并批量屏蔽命中帖数大于阈值的账号。
- 批量屏蔽默认按 2 秒间隔串行请求，可自定义请求间隔；遇到 `401/403/429/503` 会停止本轮。
- 可一键恢复当前页原始列表。

## 批量屏蔽

`0.2.2` 起，脚本面板下方提供一个可折叠的“批量屏蔽”区域。

- 必须先运行上方筛选/排序；批量屏蔽只处理当前已经显示出来的结果，不会额外扩大扫描范围。
- 关键词支持用英文逗号、中文逗号、分号、竖线或换行分隔，例如 `中转站,注册送,企业级,team,plus,公益`。
- “命中帖数>”使用严格大于逻辑：填写 `2` 时，只会候选命中标题帖子数为 `3` 或更多的作者。
- “预览账号”只做本地统计，不会请求屏蔽接口。
- “执行屏蔽”会先确认，再按设置的间隔逐个请求 `/api/block-list/add`。
- 屏蔽接口调用形式参考 [NodeSeek X](https://greasyfork.org/en/scripts/479426-nodeseek-x/code)：`POST /api/block-list/add`，请求体为 `{ "block_member_name": "用户名" }`。

## 等级过滤 Beta

`0.1.5` 起，“隐藏等级<=”会先从列表 DOM 识别作者等级；如果列表里没有等级信息，则从作者链接提取 uid，并按需请求 NodeSeek 账户信息接口补全等级。

- 能从帖子列表 DOM 中识别等级时才过滤。
- 支持识别 `.user-lv0` 到 `.user-lv15` 这类 class。
- 支持识别 `Lv 3`、`等级 Lv 3`、`等级 3`、`title="等级"` 等文本。
- DOM 识别失败时，会请求 `/api/account/getInfo/{uid}`，读取返回中的 `detail.rank`。
- `uid -> level` 会缓存 7 天，最多保留 1200 个用户等级。
- 补全请求支持自定义并发，默认 `2`，范围 `1-5`。
- 等级补全会和每页 5 秒加载冷却并行执行，尽量减少额外等待。
- 遇到 `429` 会进入全局冷却，短时间内后续批次不再继续打等级接口。
- 等级接口失败会短暂负缓存，避免同一 uid 在后续页面里反复请求。
- 没有作者 uid 或接口补全失败的帖子默认保留，避免误删正常内容。

## 低内存设计

在 4GB 内存手机上，连续抓取几十页时最容易出问题的地方不是网络请求，而是脚本长期保留大量 DOM 节点。这个项目在 `0.1.3` 后做了几项优化：

- 不长期保存每个帖子的 DOM clone 或原始 `outerHTML`，只保存排序/过滤/渲染需要的紧凑字段。
- 每抓取一页就立即过滤和去重，减少无关记录进入后续排序。
- 渲染时使用轻量结果行，减少同时持有的节点和字符串体积。
- 过滤和排序优先交给 Worker 执行，减少主线程卡顿。
- 保留中止按钮，发现页面明显变慢时可以及时停止。

低内存设备建议：

- 后续页数先从 5-10 页试起。
- 尽量配合“天内”和“用户名”过滤，减少候选帖子数量。
- 如果启用等级过滤，建议先用较小页数测试识别效果。
- 不建议在手机浏览器里一次拉取 50+ 页。

## 路线图

计划在后续版本加入：

- 关键词过滤。
- 排除词，例如 `-广告`。
- 标题模糊匹配。
- 相关性排序。
- 结果缓存，避免反复调整关键词时重新等待逐页请求。

建议的相关性策略：

- 标题完整命中给最高权重。
- 标题分词命中次高。
- 分类、作者命中作为辅助权重。
- 发布时间、浏览量、评论数只作为轻量加分，不覆盖关键词本身。
- 中文模糊匹配优先使用轻量 n-gram，不引入大型搜索库，避免移动端内存压力。

## 版本

- `0.2.2`：加入基于当前筛选结果的关键词命中账号批量屏蔽，提供折叠 UI、预览、确认、串行限速和异常状态停止保护。
- `0.2.1`：加入 IndexedDB 后续页缓存和 Worker 过滤/排序，支持失败自动回退。
- `0.2.0`：加入等级 API 全局 429 冷却、失败负缓存、轻量渲染、页面结构检测和可取消请求/等待，并删除死代码。
- `0.1.9`：等级补全并发范围调整为 `1-5`。
- `0.1.8`：等级补全会在等待下一页加载的 5 秒冷却时间内异步执行。
- `0.1.7`：等级补全支持自定义并发，遇到 429 自动暂停本轮补全。
- `0.1.6`：移除显示上限字段和结果裁剪逻辑，完整展示过滤后的排序结果。
- `0.1.5`：切换许可证到 AGPL-3.0-only，修复等级过滤，增加作者等级 API 补全与缓存。
- `0.1.4`：项目和脚本改名为 `nodeseek-sou`，加入 beta 等级过滤。
- `0.1.3`：加入低内存模式、显示上限、抓取中裁剪候选结果。
- `0.1.2`：加入发布时间过滤和用户名过滤。
- `0.1.1`：将分页抓取间隔改为 5 秒。
- `0.1.0`：初始版本，支持多页抓取并按浏览量/评论数排序。
